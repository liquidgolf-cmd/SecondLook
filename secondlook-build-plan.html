01 — Overview

What You're Building

SecondLook is a web-based digital safety companion that helps seniors confidently navigate messages, links, and payment requests. It features Nora — a calm AI guide — and an optional family dashboard. Here's the architectural overview before we dive in.

Frontend (React)

Single-page app with three zones: the user dashboard, the message check interface, and the family summary view. Calm UI, minimal alerts.

Backend (Supabase)

User auth, family connections, permission settings, and weekly summary storage. No message content stored — analysis only.

AI Core (Claude API)

Nora's analysis engine. Receives a message or link, returns structured risk assessment in JSON. Never stores conversation content.

05 — Nora's AI Core

The System Prompt That Powers Nora

This is the exact system prompt to use in your Supabase Edge Function when calling the Claude API. It defines Nora's personality, analysis framework, and required JSON output format. Copy this exactly.

Supabase Edge Function · analyze/index.ts · System Prompt
You are Nora, a calm and trusted digital safety companion for SecondLook. Your role is to analyze messages, links, or text that a user has submitted for a second opinion. You are NOT a warning system. You are a helpful assistant who notices unusual patterns and provides gentle, non-judgmental context. PERSONALITY RULES — follow these exactly: - Always calm. Never alarming. Never dramatic. - Speak in short, clear sentences. - Never shame the user. Normalize human error. - Never say: "This is a scam", "You are in danger", "High risk detected" - Never use exclamation points. - Always say: "This message includes..." not "This is a..." - Place blame on the scam design, never the user. - Always preserve autonomy — the user decides what to do. DETECTION PATTERNS — flag these when present: 1. urgency_language — "immediately", "30 minutes", "urgent", "right now" 2. payment_pressure — requests for payment, wire, Venmo, Zelle 3. crypto_request — Bitcoin, crypto, gift cards, prepaid cards 4. secrecy_request — "don't tell", "keep this between us", "don't mention" 5. authority_impersonation — FBI, IRS, bank, Medicare, Social Security 6. family_impersonation — claims to be grandchild, relative in trouble 7. threat_language — arrest, lawsuit, account closure, legal action 8. link_suspicion — mismatched domains, misspelled brands, redirect chains RISK LEVELS: - low: 0–1 flags present, content seems normal - medium: 2–3 flags, warrants a second look - high: 3+ flags, or any crypto/gift card request, or any secrecy + payment combo OUTPUT FORMAT — respond ONLY with valid JSON, no other text: { "risk_level": "low" | "medium" | "high", "nora_message": "Her exact words to the user — calm, 1-2 sentences max", "flags": [ { "type": "urgency_language", "plain_english": "This message creates a sense of urgency.", "educational_note": "Legitimate organizations rarely demand immediate action." } ], "recommended_actions": ["Call your bank directly", "Ask a trusted family member"], "pause_seconds": 0 | 5 | 9 } NORA'S MESSAGE EXAMPLES by risk level: - low: "This one looks straightforward. No unusual patterns detected." - medium: "This message includes some patterns worth a closer look." - high: "Let's take a second look. This message includes several patterns often used in scams." The pause_seconds field: return 0 for low, 5 for medium, 9 for high risk. For high risk involving crypto or gift cards, always return 9. Remember: You are backup, not a gatekeeper. The user is always in control.
💡
Mode-based sensitivity

Pass the user's mode ('gentle', 'standard', 'watchful') as a variable in the user message: "User mode: gentle. Analyze: [message]". In gentle mode, only surface high-risk results. In watchful mode, surface medium and high.

07 — Feature Specifications

Core Features — What to Build

Each feature below includes front-end behavior, back-end requirements, and Cursor prompt suggestions to accelerate development.

Message Check Interface
Core
User Experience
  • Large, inviting textarea: "Paste a message, text, or email here"
  • Single CTA button: "Get Nora's Second Look"
  • Nora avatar animates with breathing effect during analysis (1–3 seconds)
  • Result appears as a calm card — green/yellow/navy color coding (not red)
  • Pause button greys out for pause_seconds before user can dismiss
  • Optional voice: Nora speaks her message if enabled
  • Two actions: "Continue anyway" / "Ask for help" (contacts family)
Technical Requirements
  • POST to Supabase Edge Function /analyze with message text + user mode
  • Receive structured JSON from Claude — display flags in plain English
  • Log risk_event to database (metadata only, no message text)
  • If high risk + family_share_level includes alerts → trigger notification
  • Web Speech API: window.speechSynthesis for Nora's voice
  • No message text stored in database after analysis is complete
Nora Overlay + 9-Second Pause
AI Behavior
Interaction Design
  • Screen softens (brightness 90% via CSS filter) when Nora appears
  • Nora avatar fades in bottom-right corner — no bounce, no flash
  • Slow breathing animation: 2s inhale, 2s exhale (Framer Motion)
  • Risky text gets underline animation — teal, 0.4s draw from left
  • Primary action button opacity: 0.4 → 1.0 over pause_seconds duration
  • Progress ring around Nora avatar counts down the pause visually
  • User can always click "I'm sure" to bypass — no hard blocks ever
Animation Specs
  • Breathing: scale 1.0 → 1.03 → 1.0, opacity 0.9 → 0.7 → 0.9
  • Overlay fade-in: opacity 0 → 1, duration 600ms, ease-in-out
  • Screen softening: CSS filter brightness(0.9), transition 500ms
  • Underline: SVG path draw animation, stroke-dashoffset technique
  • Button enable: opacity transition, not visibility — always tappable
  • Exit: overlay fades out over 400ms, screen returns to full brightness
Three Modes — Gentle, Standard, Watchful
Core
Mode Behavior Differences
  • Gentle: Only show Nora for high risk. Low/medium = silent pass
  • Standard: Show Nora for medium + high. Low = silent green check
  • Watchful: Show Nora for all results. Extra family notification triggers
  • Mode stored in users table, respected in every analysis call
  • Pass mode to Edge Function: Claude adjusts output accordingly
  • In-app mode switcher: three cards with radio selection, saves instantly
Mode UX Copy
  • Gentle label: "Only step in when something looks high-risk."
  • Standard label: "Give me a second opinion when something looks unusual."
  • Watchful label: "Help me double-check more often."
  • Mode change confirmation: "Got it, Linda. I'll stay in the background more."
  • No dark patterns — switching to Gentle is celebrated, not discouraged
Family Dashboard + Alerts
Family
Senior Controls (Permission Panel)
  • Toggle: "Notify [Name] for high-risk events only"
  • Toggle: "Share weekly summaries with [Name]"
  • Toggle: "Private mode — no family notifications"
  • Senior sees exactly what family sees — no hidden info
  • "Shared summary active" indicator always visible to senior
  • One-tap to revoke access, no confirmation dark pattern
Family View (Read-Only)

    10 — Privacy Architecture

    Privacy as Product — Not a Feature

    These aren't just policies — they're engineering decisions. Each one should be implemented technically, not just promised in terms of service.

    🗑️

    Zero Message Storage

    Message text flows through the Edge Function to Claude and is discarded. Only flag metadata is written to the database. Enforce this in code with a database trigger that rejects any row containing message_text.

    🔐

    Row Level Security

    Every Supabase table has RLS enabled. Users can only read their own rows. Family members access only an aggregated summary view — never individual events with identifying information.

    🧠

    No Claude Memory

    Each analysis call is stateless. Claude receives no conversation history, no previous messages, and no user profile. Every check is a fresh, isolated request.

    👁️

    Visible Data Dashboard

    In Settings, show the user a live read of their data: "Messages stored: 0 · Events logged: 4 · Family connections: 1." Transparency builds trust more than any privacy policy.

    🗂️

    Delete Everything

    One-tap account deletion removes all rows across all tables via a Supabase function. No "processing period." Instant, irreversible, confirmed clearly to the user.

    🚫

    No Ads, Ever

    Encode this in your Stripe setup: subscription is the only revenue model. No analytics pixels, no third-party tracking scripts. Vercel Analytics only — privacy-first by default.
